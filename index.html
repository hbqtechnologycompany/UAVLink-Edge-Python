Tổng quan hệ thống System overview #

UAVLink-Edge là phần mềm chạy trên companion computer (thường là Raspberry Pi), đóng vai trò cầu nối giữa flight controller và cloud server qcloudstation. UAVLink-Edge runs on a companion computer (typically Raspberry Pi), bridging the flight controller and the qcloudstation cloud server.

┌─────────────────────────────────────────────────────────────────┐ │ LUỒNG DỮ LIỆU TỔNG QUAN │ └─────────────────────────────────────────────────────────────────┘ [Flight Controller] ──Serial/UART/UDP──► [UAVLink-Edge.py] Pixhawk / Custom FC Raspberry Pi │ auth_client.py │ TCP :5770 ──► HMAC Handshake │ ◄── Session Token │ │ forwarder.py │ UDP :14550 ──► MAVLink frames │ ◄── Commands (ARM/DISARM/...) │ │ heartbeat_loop │ UDP 1Hz ──► MSG_SESSION_REFRESH [qcloudstation] ◄──────────────────── [Hybrid Router] Fleet Dashboard UI Redis Pub/Sub Go server http://qcloudcontrol.com TCP :5760 (QGC) :8001 WebSocket
ℹ
Điểm mấu chốt về xác thực: auth_client.py và forwarder.py là 2 file cốt lõi. Mọi hướng tích hợp đều tái sử dụng 2 file này — chỉ thay đổi nguồn MAVLink đầu vào. Key insight: auth_client.py and forwarder.py are the two core files. Every integration route reuses them — you only change the MAVLink source input.
Giao thức xác thực (HMAC 2 lớp) Auth protocol (2-layer HMAC) #

4 bước handshake qua TCP. Hiểu rõ để debug và tích hợp đúng. 4-step TCP handshake. Essential knowledge for debugging and correct integration.

BướcMsgHướngPayloadGhi chú
10x01 AUTH_INITDrone → Server[UUID_LEN:2][UUID:var]Gửi drone UUID để server biết cần verify ai
20x02 AUTH_CHALLENGEServer → Drone[NONCE_LEN:2][NONCE:var][TIMEOUT:2]Nonce ngẫu nhiên chống replay attack
30x03 AUTH_RESPONSEDrone → Server[UUID][HMAC:32][TIMESTAMP:8][IP]HMAC ký nonce bằng combined_key
40x04 AUTH_ACKServer → Drone[RESULT:1][TOKEN:var][EXP:8][INT:2]Nhận session token, TTL thường 90–300s
🔑 Công thức HMAC (source of truth: auth_client.py)
Pythonauth_client.py → authenticate()
# Bước 1: Tạo combined key từ 2 nguồn secret combined_key_seed = (self.secret_key + self.shared_secret).encode('utf-8') combined_key = hashlib.sha256(combined_key_seed).hexdigest().encode('utf-8') # Bước 2: HMAC-SHA256 ký lên nonce từ server signature = hmac.new(combined_key, nonce, hashlib.sha256).digest() # Giải thích bảo mật: # - secret_key: lưu trong .drone_secret (kết quả từ lần đăng ký) # - shared_secret: config.yaml — cấp bởi qcloudstation # - Nonce: mỗi lần auth khác nhau → chống replay attack # - Timestamp: gửi kèm trong AUTH_RESPONSE, server từ chối nếu lệch >60s
⚠
Inconsistency trong tài liệu: README cấp cao (hệ thống) ghi formula khác: HMAC(shared_secret, "{droneID}:{nonceHex}:{timestamp}"). Đây là formula của Hybrid Router Go server, KHÔNG phải Python client. Luôn tin vào auth_client.py khi viết Python.

import time, socket, threading from pymavlink import mavutil class MAVLinkEmitter: """ Đọc dữ liệu từ hardware tùy chỉnh và phát MAVLink lên UDP local để forwarder.py nhận và gửi lên server. """ def __init__(self, output_port=14540, system_id=1): self.mav = mavutil.mavlink_connection( f'udpout:127.0.0.1:{output_port}' ) self.running = False def _read_hardware(self): # TODO: Thay bằng logic đọc sensor thực của bạn # Ví dụ: đọc IMU qua I2C, đọc GPS qua UART... return { 'roll': 0.0, 'pitch': 0.0, 'yaw': 0.0, 'lat': 10.762622, 'lon': 106.660172, 'alt': 10.0, 'battery_v': 11.8, 'battery_pct': 85 } def emit_loop(self): while self.running: d = self._read_hardware() t = int(time.time() * 1e6) # microseconds # HEARTBEAT — bắt buộc, server dùng để biết drone online self.mav.mav.heartbeat_send( mavutil.mavlink.MAV_TYPE_GROUND_ROVER, mavutil.mavlink.MAV_AUTOPILOT_GENERIC, 0, 0, mavutil.mavlink.MAV_STATE_ACTIVE ) # ATTITUDE — roll/pitch/yaw cho PFD display self.mav.mav.attitude_send( t, d['roll'], d['pitch'], d['yaw'], 0, 0, 0 ) # GPS_RAW_INT — vị trí trên bản đồ self.mav.mav.gps_raw_int_send( t, 3, int(d['lat'] * 1e7), int(d['lon'] * 1e7), int(d['alt'] * 1000), 65535, 65535, 0, 0, 255 ) # BATTERY_STATUS self.mav.mav.battery_status_send( 0, 0, mavutil.mavlink.MAV_BATTERY_FUNCTION_ALL, mavutil.mavlink.MAV_BATTERY_TYPE_LIPO, 2100, [int(d['battery_v'] * 1000)] + [65535] * 9, d['battery_pct'], -1, -1, -1, mavutil.mavlink.MAV_BATTERY_CHARGE_STATE_OK, [], 0, -1 ) time.sleep(0.1) # 10Hz def start(self): self.running = True threading.Thread(target=self.emit_loop, daemon=True).start()

Route 4 — Full Protocol Bridge (nâng cao) #

Thiết bị giao tiếp bằng giao thức riêng, không muốn dùng MAVLink. Wrap payload vào MAVLink TUNNEL message.

⚠

Chiến lược: MAVLink TUNNEL (Message ID 385)
Pythoncustom_protocol_bridge.py
from pymavlink import mavutil import json, struct class ProtocolBridge: """Wrap giao thức tùy chỉnh vào MAVLink TUNNEL message.""" PAYLOAD_TYPE_CUSTOM = 0xF0 # Tự định nghĩa def send_custom_data(self, mav_conn, data: dict): # Serialize data thành JSON bytes payload = json.dumps(data).encode('utf-8') # Pad đến 128 bytes (max TUNNEL payload) payload = payload.ljust(128, b'\x00')[:128] mav_conn.mav.tunnel_send( 0, # target_system 0, # target_component self.PAYLOAD_TYPE_CUSTOM, # payload_type (custom) len(payload), # payload_length list(payload) # payload bytes ) def receive_custom_data(self, mav_conn): msg = mav_conn.recv_match(type='TUNNEL', blocking=False) if msg and msg.payload_type == self.PAYLOAD_TYPE_CUSTOM: raw = bytes(msg.payload[:msg.payload_length]) return json.loads(raw.rstrip(b'\x00')) return None

Sub-route 5B — rc_reader.py: đọc SBUS trực tiếprc_reader.py: read SBUS directly

SBUS là giao thức serial 100kbps inverted. Pi 5 cần đảo tín hiệu (inverter circuit) hoặc dùng receiver hỗ trợ SBUS non-inverted (ExpressLRS, ELRS receiver với pin TX2). SBUS is a 100kbps inverted serial protocol. Pi 5 needs signal inversion (inverter circuit) or a receiver with non-inverted SBUS output (ExpressLRS, ELRS receivers with TX2 pin).

Pythonrc_reader.py ← file mới thêm vào repo
import serial, threading, time, logging logger = logging.getLogger("RCReader") class SBUSReader: """ Đọc SBUS từ RC receiver nối vào UART Pi 5. SBUS spec: 100000 baud, 8E2, inverted signal. Wiring: Receiver SBUS out ──[100Ω]──[NPN transistor inverter]──► /dev/ttyAMA2 (hoặc dùng ELRS receiver với non-inverted SBUS / CRSF output) Non-inverted alternatives (không cần inverter): - ExpressLRS receiver với CRSF → dùng CRSFReader thay thế - FrSky R-XSR với pad "SPORT" đã invert sẵn """ SBUS_FRAME_LEN = 25 SBUS_HEADER = 0x0F SBUS_FOOTER = 0x00 def __init__(self, port="/dev/ttyAMA2", callback=None): self.port = port self.callback = callback # fn(channels: list[int]) — 16 channels, 172–1811 self.channels = [992] * 16 # midpoint default self.failsafe = False self.running = False self._ser = None self._buf = bytearray() def start(self): try: self._ser = serial.Serial( self.port, baudrate=100000, bytesize=serial.EIGHTBITS, parity=serial.PARITY_EVEN, stopbits=serial.STOPBITS_TWO, timeout=0.1 ) self.running = True threading.Thread(target=self._read_loop, daemon=True).start() logger.info(f"SBUS reader started on {self.port}") except Exception as e: logger.error(f"SBUS open failed: {e}") def _read_loop(self): while self.running: try: data = self._ser.read(64) if not data: continue self._buf.extend(data) self._parse_frames() except Exception as e: logger.warning(f"SBUS read error: {e}") time.sleep(0.01) def _parse_frames(self): while len(self._buf) >= self.SBUS_FRAME_LEN: # Tìm header byte 0x0F idx = self._buf.find(self.SBUS_HEADER) if idx == -1: self._buf.clear(); return if idx > 0: del self._buf[:idx] # strip garbage if len(self._buf) < self.SBUS_FRAME_LEN: return frame = self._buf[:self.SBUS_FRAME_LEN] del self._buf[:self.SBUS_FRAME_LEN] # Decode 16 channels từ 11-bit packed values b = frame[1:23] ch = [0] * 16 bit_buf = int.from_bytes(b, 'little') for i in range(16): ch[i] = (bit_buf >> (i * 11)) & 0x7FF self.failsafe = bool(frame[23] & 0x08) self.channels = ch if self.callback: self.callback(ch) def get_channels(self) -> list: return self.channels.copy() def sbus_to_pwm(self, sbus_val: int) -> int: # SBUS 172–1811 → PWM 1000–2000 µs return int(1000 + (sbus_val - 172) * 1000 / (1811 - 172))
Pythonrc_reader.py — CRSFReader cho ExpressLRS (không cần inverter)
class CRSFReader: """ Đọc CRSF protocol từ ExpressLRS / TBS Crossfire receiver. Không cần inverter. Baud: 420000. UART TX của receiver → Pi RX. Ưu điểm CRSF vs SBUS: - Không cần inverter circuit - Tốc độ cao hơn (416 fps vs 112 fps) - Có link statistics (RSSI, SNR, LQ) — gửi lên cloud được """ CRSF_SYNC = 0xC8 CRSF_CHANNELS = 0x16 CRSF_LINK_RX = 0x14 def __init__(self, port="/dev/ttyAMA2", callback=None, link_callback=None): self.port = port self.callback = callback # fn(channels: list[int]) — 16 ch, 172–1811 self.link_callback = link_callback # fn(rssi, snr, lq) — link quality self.running = False self._ser = None def start(self): self._ser = serial.Serial(self.port, baudrate=420000, timeout=0.1) self.running = True threading.Thread(target=self._read_loop, daemon=True).start() logger.info(f"CRSF reader started on {self.port}") def _read_loop(self): while self.running: byte = self._ser.read(1) if not byte or byte[0] != self.CRSF_SYNC: continue length = self._ser.read(1) if not length: continue frame = self._ser.read(length[0]) if len(frame) < length[0]: continue frame_type = frame[0] payload = frame[1:-1] # strip CRC if frame_type == self.CRSF_CHANNELS and len(payload) == 22: ch = [0] * 16 bit_buf = int.from_bytes(payload, 'little') for i in range(16): ch[i] = (bit_buf >> (i * 11)) & 0x7FF if self.callback: self.callback(ch) elif frame_type == self.CRSF_LINK_RX and len(payload) >= 8: rssi = -payload[0] snr = payload[2] - 128 # signed lq = payload[3] # link quality % if self.link_callback: self.link_callback(rssi, snr, lq)

Cameras supported: Pi Camera Module 3: libcamera-vid (tốt nhất, hardware H.264) USB webcam: ffmpeg v4l2 (fallback, CPU encode) """ def __init__(self, config): self.cfg = config.get('video', {}) self.proc = None self.running = False # Lấy từ config.yaml — xem section video bên dưới self.source = self.cfg.get('source', 'picamera') self.rtsp_url = self.cfg.get('mediamtx_rtsp', 'rtsp://45.117.171.237:8554') self.drone_id = self.cfg.get('stream_name', 'drone1') self.width = self.cfg.get('width', 1280) self.height = self.cfg.get('height', 720) self.fps = self.cfg.get('fps', 30) self.bitrate = self.cfg.get('bitrate_kbps', 2000) def _build_cmd(self): target = f"{self.rtsp_url}/{self.drone_id}" if self.source == 'picamera': # Giới hạn quan trọng — Sub-route 5B: RC data lên qcloudstation chỉ dùng để monitor (xem stick position trên dashboard). Không thể dùng để điều khiển motor/servo từ Pi nếu không có FC — Pi không có real-time OS, jitter ~1ms sẽ làm servo rung. Chỉ dùng 5B cho robot mặt đất, camera car, hoặc thiết bị không yêu cầu control loop nhanh. Important limitation — Sub-route 5B: RC data to qcloudstation is for monitoring only (view stick position on dashboard). Without a FC, you cannot drive motors/servos from Pi — Pi has no RTOS, ~1ms jitter will cause servo chatter. Use 5B only for ground robots, camera cars, or devices that don't need fast control loops.

🤖 AI Agent — Quy tắc sửa code #
RULE 01 — Không bao giờ thay đổi HMAC formula

Công thức trong auth_client.py → authenticate(): SHA256(secret_key + shared_secret) rồi HMAC-SHA256(combined_key, nonce) là giao kèo với server. Thay đổi sẽ khiến auth fail hoàn toàn. Nếu task yêu cầu "sửa auth", hỏi lại có phải server đã thay đổi spec không.

RULE 02 — heartbeat_loop() phải duy trì 1Hz

forwarder.py → heartbeat_loop() gửi MSG_SESSION_REFRESH mỗi 1 giây. Đây là cơ chế NAT traversal. Nếu dừng hoặc chậm hơn 10 giây, server mất endpoint UDP của drone. Không được sleep lâu hơn 1 giây trong hàm này.

RULE 03 — Session token phải tồn tại trước khi gửi MAVLink

uplink_loop() check if self.auth_client.session_token: trước khi gửi. Đây là thiết kế đúng — không được bỏ check này. Nếu thêm route UDP mới, phải có cùng check này.

RULE 04 — Config mới phải có giá trị default

Khi thêm config section vào config.yaml và đọc trong code, luôn dùng .get('key', default_value). Không dùng ['key'] trực tiếp — sẽ crash nếu người dùng không có key đó trong file config cũ.

RULE 05 — Thread mới phải là daemon

Mọi thread thêm vào phải có daemon=True: threading.Thread(target=..., daemon=True).start(). Thiếu điều này sẽ khiến Ctrl+C không tắt được process vì thread không chịu thoát.

RULE 06 — stats_lock khi truy cập self.stats

forwarder.py có self.stats_lock. Mọi đọc/ghi self.stats phải dùng with self.stats_lock:. Web server đọc stats từ thread khác — race condition sẽ xảy ra nếu bỏ lock.

RULE 07 — Không ghi credentials vào code

uuid, shared_secret phải đọc từ config.yaml, không hardcode trong Python. .drone_secret là file runtime, không commit lên git (đã có trong .gitignore).

RULE 08 — File mới thêm phải được import trong main.py

Khi tạo module mới (mavlink_emitter.py, ros2_bridge.py...), phải wiring vào main.py. Module tự chạy khi import là anti-pattern — dùng class với method start() và gọi từ main().
